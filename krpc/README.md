# Overview. What is kRPC

kRPC is a custom bidirectional RPC protocol within the kotlinx-rpc library. 
It is one of two protocol implementations (the other being gRPC). 
kRPC uses message-based communication over pluggable transports (canonical: Ktor WebSockets).

## Build & Test

Use `running_gradle_tests` skill:
- Specific krpc module: `:krpc:krpc-core:jvmTest`, `:krpc:krpc-client:jvmTest`, `:krpc:krpc-test:jvmTest`
- All targets (when platform-specific code changed): `:krpc:krpc-test:allTests`
- Single test: `:krpc:krpc-test:jvmTest --tests "TestClass.testMethod"`
- Include stress tests (excluded by default): `:krpc:krpc-core:jvmTest -PstressTests=true`
- Compatibility tests (separate test modules): `:tests:krpc-compatibility-tests:jvmTest`, `:tests:krpc-protocol-compatibility-tests:jvmTest`

## Compatibility Tests (in `/tests/`)

- **krpc-compatibility-tests** -- loads old/new API versions via separate ClassLoaders, tests cross-version client↔server communication
- **krpc-protocol-compatibility-tests** -- template-generated code for each protocol version, matrix-tests all version pairs with dynamic JUnit 5 test factories
